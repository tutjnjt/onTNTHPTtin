ĐỊA CHỈ IP
🔢
IPv4 vs IPv6
  • IPv4: 32 bit, dạng a.b.c.d (VD: 192.168.1.1)
  • IPv4: ~4.3 tỉ địa chỉ, đang cạn kiệt
  • IPv6: 128 bit, dạng hex (VD: 2001:db8::1)
  • IPv6: gần như vô hạn địa chỉ
🏠
IP riêng vs IP công cộng
  • IP riêng: 192.168.x.x, 10.x.x.x – chỉ trong LAN
  • IP công cộng: do ISP cấp, định tuyến trên Internet
  • NAT: chuyển IP riêng ↔ IP công cộng
  • IP tĩnh: cố định | IP động: DHCP cấp tự động
🎯 Mẹo thi: Hay nhầm Hub và Switch – nhớ: Switch = Thông minh (gửi đúng cổng). DNS = "danh bạ" Internet. 192.168.x.x = IP nội bộ (KHÔNG ra được Internet trực tiếp).

CSS – SELECTOR & THUỘC TÍNH

/* Tag selector – áp dụng cho TẤT CẢ thẻ p */ p { color: blue; font-size: 16px; } /* Class selector (.) – áp dụng cho class="noi-bat" */ .noi-bat { font-weight: bold; color: red; } Dùng với: <p class="noi-bat"> /* ID selector (#) – áp dụng cho id="tieu-de" (duy nhất) */ #tieu-de { font-size: 24px; text-align: center; } Dùng với: <h1 id="tieu-de"> /* Ưu tiên: Inline > #id > .class > tag */
CSS – MÔ HÌNH HỘP (BOX MODEL)
margin · border · padding · content

KỸ THUẬT ĐỌC CODE
📖 Chiến lược đọc code vận dụng:
1️⃣ Xác định INPUT (tham số / biến đầu vào)
2️⃣ Theo dõi từng vòng lặp với giá trị cụ thể nhỏ

CÂU LỆNH SQL CƠ BẢN
SELECT – Lệnh truy vấn dữ liệu quan trọng nhất
-- Lấy TẤT CẢ cột và hàng SELECT * FROM HocSinh; -- Lấy một số cột cụ thể SELECT HoTen, DiemTB FROM HocSinh; -- Lọc có điều kiện (WHERE) SELECT * FROM HocSinh WHERE DiemTB >= 8.0; -- Kết hợp nhiều điều kiện SELECT * FROM HocSinh WHERE Lop = '12A1' AND DiemTB >= 6.5; -- Sắp xếp kết quả (ORDER BY) SELECT HoTen, DiemTB FROM HocSinh ORDER BY DiemTB DESC; -- DESC=giảm, ASC=tăng (mặc định) -- Tìm kiếm chuỗi (LIKE) SELECT * FROM HocSinh WHERE HoTen LIKE 'Nguyễn%'; -- % = nhiều ký tự bất kỳ | _ = MỘT ký tự bất kỳ
HÀM TỔNG HỢP (AGGREGATE FUNCTIONS)
HàmChức năngVí dụ
COUNT(*)Đếm số bản ghiSELECT COUNT(*) FROM HocSinh;
SUM(cột)Tính tổngSELECT SUM(DiemToan) FROM HocSinh;
AVG(cột)Tính trung bìnhSELECT AVG(DiemTB) FROM HocSinh;
MAX(cột)Giá trị LỚN NHẤTSELECT MAX(DiemTB) FROM HocSinh;
MIN(cột)Giá trị NHỎ NHẤTSELECT MIN(DiemTB) FROM HocSinh;
⚠️ Hay nhầm: AVG ≠ MAX. AVG = trung bình, MAX = lớn nhất. ORDER BY không có từ khóa = mặc định ASC (tăng dần). Muốn giảm dần = phải ghi DESC.
🎯 Mẹo thi: Phần II câu CSDL thường cho bảng mẫu và đoạn SQL rồi hỏi đúng/sai. Đọc kỹ: WHERE điều kiện, ORDER BY ASC/DESC, tên hàm tổng hợp. Lỗi hay gặp: nhầm ASC/DESC, nhầm AVG với MAX/COUNT.
